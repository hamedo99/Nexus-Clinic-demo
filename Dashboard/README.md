# نظام عيادة نيكسوس (NexusClinic OS)
## نظام إدارة وتنسيق المواعيد الطبية عالي الدقة (نسخة العراق)

### 1. الملخص التنفيذي
نظام عيادة نيكسوس هو نظام بيئي مزدوج الواجهة مصمم لسد الفجوة بين سهولة الوصول الرقمي للمرضى وكفاءة الإدارة في العيادات. يعتمد النظام على بنية حديثة موجهة بالأحداث (Event-Driven)، ويوفر تزامنًا سلسًا بين بوابة حجز المرضى العامة ولوحة تحكم إدارية عالية الأداء ومترجمة بالكامل للعربية. الشعار الأساسي هو "التعقيد في الخلفية، والبساطة على الشاشة"، لضمان تمكين الموظفين غير التقنيين في العيادات العراقية من إدارة جداول المواعيد المزدحمة دون أي تعقيد.

### 2. المكدس التقني والبنية (Technical Stack)
يعتمد النظام نهج "Serverless-First" لضمان وقت تشغيل بنسبة 99.9% وزمان انتقال منخفض (Low Latency).

- **إطار العمل (Framework):** Next.js 15 (App Router) للعرض الهجين (SSR/ISR) لضمان تحسين محركات البحث (SEO) للمرضى والسرعة للوحة التحكم.
- **اللغة (Language):** TypeScript (Strict Mode) لضمان سلامة النوع وتقليل أخطاء وقت التشغيل.
- **محرك الوقت الفعلي (Real-time Engine):** Supabase (PostgreSQL + Realtime) مع Row Level Security (RLS).
- **نواة التقويم (Calendar Core):** FullCalendar Premium مع غلاف React مخصص لتحسين العرض من اليمين لليسار (RTL).
- **واجهة المستخدم (UI/UX):** Tailwind CSS + Radix UI (via Shadcn/UI) لتصميم معياري وسهل الوصول.

### 3. الميزات عالية الدقة
#### أ. لوحة تحكم السكرتارية "بدون منحنى تعلم"
- **إدارة الحالة البصرية:** كتل مواعيد ملونة (مؤكد، معلق، لم يحضر، تم الدخول).
- **إجراءات ذرية (Atomic Actions):** السحب والإفلات لإعادة الجدولة مع التحقق التلقائي.
- **تحديثات واجهة المستخدم المتفائلة (Optimistic UI):** استجابة فورية للواجهة.

#### ب. التوطين للسياق العراقي
- **منطق التقويم الإقليمي:** دعم أسبوع العمل العراقي (الأحد-الخميس) وزر "عطلة رسمية/إغلاق طارئ".
- **تكامل الاتصالات:** خطافات API لبوابات الرسائل المحلية (Zain, Asiacell) وتذكيرات WhatsApp.
- **طبقة بيانات ثنائية اللغة:** واجهة عربية مع دعم إدخال بيانات ثنائي اللغة.

#### ج. حل النزاعات والمنطق المتقدم
- **توليد الخانات الديناميكي:** خوارزمية لتوليد خانات الحجز بناءً على "مدة الجلسة".
- **الحجز الشبحي (Ghost Booking Prevention):** قفل الخانات لمدة 5 دقائق لمنع الحجز المزدوج.

### 4. أفضل الممارسات الهندسية
- **قابلية التوسع (Scalability):** مخطط قاعدة بيانات منظم لدعم عيادات متعددة الأطباء.
- **تطبيق ويب تقدمي (PWA Ready):** قابل للتثبيت على سطح المكتب أو الجهاز اللوحي.
- **الأمان:** تشفير بيانات المرضى (PII).

### 5. خارطة طريق التنفيذ (Implementation Roadmap)
- **المرحلة 1:** مخطط قاعدة البيانات ومنطق Supabase Realtime.
- **المرحلة 2:** تخصيص FullCalendar ومكونات واجهة المستخدم العربية (RTL).
- **المرحلة 3:** بوابة المرضى وتكامل API العام.
- **المرحلة 4:** محرك إشعارات WhatsApp/SMS واختبار الضغط.
